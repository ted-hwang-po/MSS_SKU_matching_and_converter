# MSS SKU 매칭 및 발주 파일 변환기 — 요구사항 정의서

## 1. 개요

### 1.1 목적
구글 시트(또는 엑셀/CSV)에 있는 상품 발주 수량 목록을 기준으로, **시스템 업로드용** 파일과 **브랜드 송부용** 파일 2개의 `.xlsx` 파일을 자동 생성한다.

### 1.2 배경
- 자체 상품코드(UID)는 **상품 종류**만 구분하며, 색상·옵션 등 최종 SKU를 반영하지 못함
- 바코드(88코드)는 **최종 옵션 단위**의 고유 식별자 (브랜드사 제공)
- 상품코드와 바코드가 1:1 매칭되지 않으므로, 별도 매칭 데이터셋과 옵션 정보를 결합해야 함
- 현재 이 작업을 수작업으로 수행 중이며, 이를 자동화하는 것이 목표

## 2. 용어 정의

| 용어 | 설명 | 샘플 데이터 내 컬럼명 |
|------|------|----------------------|
| **바코드** | 상품 최종 옵션 단위 고유 식별자 (13자리) | `88코드`, `바코드` |
| **상품코드 (UID)** | 자체 상품코드, 옵션 미반영 | `상품코드` (⚠️ `1P 상품코드`, `기존 위탁 상품코드 UID`는 사용하지 않음) |
| **스타일번호** | UID와 1:1 매칭되는 별도 값 | `스타일번호` |
| **발주수량** | 오프라인 발주 수량 | `수량(오프)` (⚠️ `발주수량(Total)`이 아님) |
| **사이즈유형** | 옵션 유무 및 옵션 그룹 구분자 | `사이즈유형` (옵션 없는 상품: `MF3`) |
| **옵션** | 색상, 사이즈 등 상품 내 세부 구분 | `Size01` ~ `Size30` |
| **공급가** | VAT 제외 공급 단가 | `공급가(VAT 제외)` |
| **정상가** | VAT 포함 정상 판매가 | `정상가(VAT 포함)` |
| **상시행사가** | VAT 포함 상시 행사가 | `상시행사가(VAT 포함)` |

## 3. 입력 데이터

### 3.1 총 발주 수량 데이터셋 (파일 1)
사용자가 업로드하는 입력 파일. 브랜드별 전체 상품의 발주 정보가 담겨 있음.

| 주요 컬럼 | 설명 | 비고 |
|-----------|------|------|
| `브랜드명` | 브랜드 필터링 기준 | |
| `88코드` | 바코드 (13자리) | 매칭 키 |
| `상품번호` | 위탁 상품번호 | UID가 아님, 참고용 |
| `상품명` | 상품 전체 이름 (옵션 포함) | 옵션 매핑에 활용 |
| `공급가(VAT 제외)` | 공급 단가 | 소수점 발생 가능 |
| `정상가(VAT 포함)` | 정상 판매가 | |
| `상시행사가(VAT 포함)` | 상시 행사가 | |
| `수량(오프)` | **오프라인 발주 수량** | 이 값을 사용 |

- 샘플: 30개 컬럼, 바코드 기준 행 구성 (옵션별 별도 행)

### 3.2 상품코드-바코드 매칭 데이터셋 (파일 2)
바코드와 자체 상품코드(UID), 스타일번호를 매핑하는 참조 데이터. 사용자가 업로드.

| 주요 컬럼 | 설명 | 비고 |
|-----------|------|------|
| `상품코드` | 자체 상품코드 (UID) | 출력 파일의 UID |
| `스타일번호` | UID와 1:1 대응 | 단일 옵션: 바코드와 동일, 다중 옵션: 텍스트 식별자 (예: `GLOW_COVER_CUSHION`) |
| `바코드` | 88코드와 동일 | 파일 1과 매칭 키 |
| `상품명` | 상품명 | |
| `옵션명` | 옵션 상세명 | |

- ⚠️ 첫 번째 행이 빈 행일 수 있음 (헤더가 2번째 행부터 시작)
- ⚠️ `1P 상품코드`, `기존 위탁 상품코드 UID` 컬럼은 사용하지 말 것

### 3.3 상품별 옵션 정보 (파일 3)
옵션이 있는 상품의 사이즈유형 코드와 옵션 목록을 정의. 사용자가 업로드.

| 주요 컬럼 | 설명 | 비고 |
|-----------|------|------|
| `사이즈유형` | 사이즈유형 코드 | 예: `SD2` |
| `사이즈유형명` | 상품명 기반 매칭용 텍스트 | 자연어 유사도 매칭에 사용 |
| `Size01` ~ `Size30` | 옵션 값 목록 | 순서대로 채워짐, 빈 값은 옵션 없음 |

## 4. 핵심 처리 로직

### 4.1 브랜드 필터링
1. 사용자가 특정 브랜드를 선택
2. 파일 1에서 해당 `브랜드명`으로 필터링

### 4.2 바코드 → 상품코드(UID) 매칭
1. 파일 1의 `88코드`를 키로 파일 2의 `바코드`와 매칭
2. 매칭된 행에서 `상품코드`(UID)와 `스타일번호`를 가져옴

### 4.3 옵션 유무 판별
1. **동일한 상품코드(UID)에 바코드가 2개 이상** 매핑되면 → 옵션 있는 상품
2. 바코드가 1개만 매핑되면 → 옵션 없는 상품

### 4.4 옵션 매핑 (옵션 있는 상품)
1. 파일 1의 `상품명`과 파일 3의 `사이즈유형명`을 **자연어 유사도 매칭**
   - 예: 상품명 `체이싱래빗 올어바웃 글로우 커버쿠션 # 쿨 라이트` → 사이즈유형명에서 `체이싱래빗 올어바웃 글로우 커버쿠션`을 포함하는 행 탐색
2. 매칭된 행의 `사이즈유형` 코드를 가져옴 (예: `SD2`)
3. 상품명에서 옵션 부분을 추출하여 `Size01`, `Size02`, ... 중 어떤 슬롯에 해당하는지 결정
   - 예: `# 쿨 라이트` → `Size01`에 해당, `# 웜 내추럴` → `Size02`에 해당

### 4.5 옵션 없는 상품 처리
- `사이즈유형` = `MF3` (고정값)
- `사이즈(Size)` = `FREE` (고정값)
- 발주수량은 `Size01` 슬롯에 전량 배치

### 4.6 가격 정보 처리
- 시스템 업로드용 파일의 가격은 **정수로 반올림**
  - `계획 정상가` ← 파일 1의 `정상가(VAT 포함)` (정수 반올림)
  - `계획 원가` ← 파일 1의 `공급가(VAT 제외)` (정수 반올림)
- 브랜드 송부용 파일의 가격도 정수 반올림
- `총 발주금액(VAT 제외)` = `공급가(VAT 제외)` × `발주수량(오프)`

## 7. 자연어 옵션 매핑 상세

옵션이 있는 상품의 경우, 상품명에서 옵션을 식별하는 로직이 필요하다.

### 매핑 방식
1. 파일 1의 `상품명`에서 기본 상품명과 옵션명을 분리
   - 일반적으로 `#` 기호 앞뒤로 분리 (예: `체이싱래빗 올어바웃 글로우 커버쿠션 # 쿨 라이트`)
2. 파일 3의 `사이즈유형명`과 기본 상품명을 유사도 비교
3. 매칭된 행의 `Size01` ~ `Size30` 값과 옵션명을 비교하여 슬롯 번호 결정

### 예시
```
상품명: "체이싱래빗 올어바웃 글로우 커버쿠션 # 쿨 라이트"
  → 기본 상품명: "체이싱래빗 올어바웃 글로우 커버쿠션"
  → 옵션명: "# 쿨 라이트" (또는 "#쿨 라이트")
  → 사이즈유형명 매칭: "체이싱래빗 올어바웃 글로우 커버쿠션 #쿨 라이트" (부분 일치)
  → 사이즈유형: SD2
  → Size01="#쿨 라이트", Size02="#웜 내추럴"
  → "# 쿨 라이트" ≈ "#쿨 라이트" → Size01 슬롯 매칭
```

### 매칭 알고리즘: 3단계 Fallback (LLM 미사용)
1. **Level 1 — 정규화 후 정확 매칭**: 모든 공백·특수문자 차이를 제거한 뒤 정확 비교
2. **Level 2 — 정규화 후 포함관계 매칭**: 기본 상품명이 사이즈유형명에 포함되는지 확인
3. **Level 3 — RapidFuzz 유사도 매칭**: `token_sort_ratio` 기반 퍼지 매칭 (임계값 80%)
4. **매칭 실패**: 자동 추정하지 않고 사용자에게 경고 표시

### 정규화 시 고려사항
- 공백 차이 무시 (`# 쿨 라이트` vs `#쿨 라이트`)
- `#` 앞뒤 공백 통일 (`# 쿨` → `#쿨`)
- 전각/반각 괄호 통일
- 연속 공백 제거
- 부분 문자열 포함 관계 확인

## 8. 기능 B: 발주 파일 통합 (Merge)

### 8.1 배경
통합된 "총 발주 수량 데이터셋"이 없고, **날짜별·브랜드별로 개별 파일**이 존재하는 경우가 있음.
이 파일들을 하나로 합쳐 기능 A(SKU 매칭 및 변환)의 입력 파일로 사용할 수 있어야 함.

### 8.2 입력
- 여러 개의 발주 파일 (`.xlsx` / `.csv`, 복수 선택)
  - 각 파일에는 최소한 `88코드`, `상품명`, `수량(오프)`, 가격 컬럼이 포함
  - 파일마다 날짜·브랜드가 다를 수 있음
- (선택) 상품코드-바코드 매칭 데이터셋 (파일 2)
  - 업로드하면 `상품번호`(1P 상품코드) 컬럼을 자동으로 채움
  - 업로드하지 않으면 `상품번호` 컬럼은 비워둠

### 8.3 출력
통합된 단일 `.xlsx` 파일. 컬럼 구성:

| 컬럼명 | 설명 | 출처 |
|--------|------|------|
| `지정입고일` | 납품/입고 예정일 | 각 파일에서 추출 또는 파일명에서 유추 |
| `브랜드명` | 브랜드 이름 | 각 파일의 `브랜드명` 컬럼 |
| `88코드` | 바코드 | 각 파일의 `88코드` 컬럼 |
| `상품번호` | 상품코드 (UID) | 파일 2 매칭 (선택) |
| `상품명` | 상품 이름 | 각 파일의 `상품명` 컬럼 |
| `공급가(VAT 제외)` | 공급 단가 | 각 파일의 가격 컬럼 |
| `정상가(VAT 포함)` | 정상 판매가 | 각 파일의 가격 컬럼 |
| `상시행사가(VAT 포함)` | 상시 행사가 | 각 파일의 가격 컬럼 |
| `발주수량(오프)` | 오프라인 발주 수량 | 각 파일의 `수량(오프)` 컬럼 |

### 8.4 처리 로직
1. 사용자가 여러 발주 파일을 복수 선택하여 업로드
2. 각 파일에서 필수 컬럼을 추출 (컬럼명 자동 인식)
3. 모든 파일의 데이터를 하나로 병합 (행 단위 concat)
4. 동일 88코드가 여러 파일에 있으면 각각 별도 행으로 유지 (중복 제거 안 함)
5. (선택) 파일 2가 업로드된 경우, 88코드 → 1P 상품코드 매칭하여 `상품번호` 컬럼 채움
6. 통합된 `.xlsx` 파일 저장

### 8.5 GUI 설계
기존 앱에 **탭** 또는 **모드 전환**으로 추가:
- "기능 A: SKU 매칭 변환" (기존)
- "기능 B: 발주 파일 통합" (신규)

기능 B 화면:
- 발주 파일 복수 선택 (파일 추가/제거)
- (선택) 상품코드-바코드 매칭 파일 선택
- 저장 위치 선택
- "통합 실행" 버튼
- 결과 로그

## 9. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| **처리 시간** | 전체 처리가 1분 이내 완료 |
| **입력 형식** | `.xlsx` 또는 `.csv` |
| **출력 형식** | `.xlsx` (2개 파일) |
| **양식 준수** | 샘플 데이터 파일의 양식(시트 구조, 헤더 행 위치, 컬럼 순서)을 그대로 따름 |
| **템플릿 유지** | 시스템 업로드용 파일의 상단 메타 행(6행), 헤더, 서브헤더 구조 유지 |

## 9. 엣지 케이스 및 주의사항

1. **파일 2의 빈 첫 행**: 헤더가 2번째 행부터 시작할 수 있음 → 빈 행 스킵 처리 필요
2. **가격 소수점**: 파일 1의 공급가가 소수점일 수 있음 (예: `6954.545454...`) → 정수 반올림
3. **옵션명 공백 차이**: `# 쿨 라이트` vs `#쿨 라이트` → 공백 정규화 후 비교
4. **UID 중복**: 옵션 있는 상품은 동일 UID에 여러 바코드 → 시스템 업로드용에서는 UID당 1행, 브랜드 송부용에서는 바코드당 1행
5. **스타일번호 형태**: 단일 옵션 상품은 바코드(숫자)가 스타일번호, 다중 옵션 상품은 텍스트 식별자 (예: `GLOW_COVER_CUSHION`)
6. **상품번호 vs UID 혼동 방지**: 파일 1의 `상품번호`(위탁 코드)와 파일 2의 `상품코드`(UID)는 다른 값
