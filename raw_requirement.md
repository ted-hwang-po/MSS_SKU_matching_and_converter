## 목적
- 구글 시트에 있는 상품 발주 수량 목록을 기준으로, 각 브랜드에 전달할 최종 발주수량을 원하는 형식에 맞춰 엑셀 파일로 생성한다.

## 배경
- 구글 시트에는 전체 상품(컬러 등 옵션) 별 바코드 정보 및 최종 발주하고자 하는 수량이 기재되어 있음.
- 자체적으로 구축한 상품코드는 최종 SKU를 반영하지 못하고, 상품 종류만 구분하고 있음.
    - 상품코드 내 옵션이 다양한 경우, 별도의 컬럼(이하 옵션코드)에 별도로 정해진 값으로 구분하고 있음.
    - ex. 디올 브랜드의 A 립스틱이라는 상품이 있을 때, 해당 상품의 코드는 AAA로 정해져있음. 립스틱의 컬러(레드, 핑크, 벽돌색 등)는 상품코드에 녹아있지 않고 별도 컬럼 (ex. size)에 코드 (ex. 00, 01..)로 기록되어야 함
- 자체 상품코드와 상품의 바코드는 1:1 매칭이 되지 않는 상태임.
    - 상품코드: 상품 단위, 옵션 및 색상 등 정보 반영되지 않음.
    - 바코드: 상품의 최종 옵션 단위, 실제 상품 패키지에 인쇄된 정보로, 브랜드사가 제공하는 정보

## 활용 데이터
1. 총 발주 수량 데이터셋 (바코드/88코드 만 있고 상품코드는 없음)
2. 상품코드-바코드 매칭 데이터셋 (상품코드와 바코드를 매핑할 수 있음)
3. 상품별 옵션 정보 (상품코드별 옵션 정보를 매핑할 수 있음)

## 결과물
4. 시스템업로드용 최종 파일
5. 브랜드송부용 최종 파일

## 용어 및 활용해야하는 컬럼
1. 바코드: 샘플데이터 내 "88코드" 혹은 "바코드"
2. 발주수량: 샘플데이터 내 "수량(오프)", 발주수량(total)이 아님
3. 상품명: 상품의 이름
4. 공급가(VAT 제외)
5. 정상가(VAT 제외)
6. 상시행사가(VAT 포함)
7. 상품코드: 샘플데이터 내 "1P 상품코드", "기존 위탁 상품코드 UID"는 사용하지 말아야 함, UID와 동일
8. 스타일번호: 1P 상품코드와 1:1 매칭되는 별도의 값
9. 사이즈유형: 옵션이 있는지 없는지 정보, 상품별 옵션 정보에서 특정 레코드를 가리키는 구분자



## 상세 사용자의 업무 프로세스 (JTBD)
0. 상품코드가 동일한데 바코드가 2개 이상인 경우 옵션이 있는 상품이라고 판단
1. 총 발주 수량 데이터셋에서 브랜드별로 필터링
2. 브랜드 별 상품 목록을 브랜드 송부용 파일에 업데이트
3. 상품코드-바코드 매칭 데이터셋을 이용해서 바코드 별 상품코드 입력
4. 상품코드와 바코드를 비교해서 옵션이 있는지 체크
5. 옵션 번호는 상품명을 이용해 상품별 옵션 정보 시트에서 '사이즈유형' 컬럼 값을 찾아서 매핑 및 어떤 옵션이 해당하는지 매핑 
    - 자연어 매핑, ex. 상품명: '체이싱래빗 올어바웃 글로우 커버쿠션 # 쿨 라이트' 면, 상품별 옵션 정보의 '사이즈유형명' 컬럼에서 '체이싱래빗 올어바웃 글로우 커버쿠션 # 쿨 라이트'와 유사한 상품을 찾고, 같은 row에 있는 옵션을 상품명으로 유추 = '# 쿨 라이트' 인지 '# 웜 내추럴' 인지
6. 나머지 컬럼을 총 발주 수량 데이터셋에서 찾아서 매핑
    - 옵션이 없는 상품의 사이즈유형 값은 'MF3'
7. 시스템 업로드용 파일에 새로운 탭(이름: Barcode 업로드(옵션))을 만들어서 스타일번호, UID, 사이즈(옵션명과 동일하므로 상품별 옵션 정보 데이터셋의 size01, size02 등 컬럼에 있는 값을 사용, 옵션이 없는 상품인 경우 항상 'FREE' 로 입력), 바코드 정보 입력
    1. 해당 탭은 '매입상품 구매오더 업로드 양식' 탭에 있는 데이터들의 실제 상품 sku 단위 유니크 메타데이터

## 사용자 시나리오
1. 사용자는 최종 발주 수량이 담긴 데이터를 구글 시트가 아니라 .csv 혹은 엑셀 파일로 업로드(혹은 로컬에 저장)한 상태이다.
2. 이 때 특정 브랜드를 선택하면, 최종 발주 수량 데이터에서 해당 브랜드로 필터링한 뒤, 상품코드 및 옵션 별 최종 발주수량이 입력된 파일(.csv, .xlsx)을 확인할 수 있다.
 

## 구현 제약사항
1. 사용자가 .xlsx 파일을 업로드하면 2개의 .xlsx 가 생성되어야함
2. 모든 작업이 1분 안에 종료되어야 함
3. 반드시 샘플 데이터 파일들의 양식을 따를 것
