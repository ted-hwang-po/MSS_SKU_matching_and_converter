# MSS SKU 매칭 및 발주 파일 변환기 — 작업계획서

## 1. 제약사항

### 1.1 보안
고객사의 보안 정책상, 발주 데이터(가격, 수량, 바코드 등)를 **외부 서버에 업로드하는 것이 제한**될 수 있음.

- 파일 데이터가 **사용자의 로컬 환경 밖으로 나가지 않을 것**
- 별도 서버 인프라가 불필요할 것

### 1.2 사용자 환경
- **Windows PC**
- **비개발자** — Python, 터미널 등 개발 환경 사용 불가
- 설치 과정이 최소화되어야 하며, **GUI 필수**

### 권장안: **D. Python GUI 데스크톱 앱 (.exe 배포)**

**선정 이유:**
1. **더블클릭으로 실행** — `.exe` 파일 하나로 설치 없이 바로 사용
2. **데이터가 로컬을 벗어나지 않음** — 네트워크 통신 일절 없음
3. **GUI 제공** — 파일 선택 대화상자, 브랜드 드롭다운, 진행 상태 표시
4. **Excel 처리 안정성** — Python `openpyxl`/`pandas`로 템플릿 양식 정확히 유지
5. **PyInstaller로 단일 .exe 빌드** — Python 미설치 환경에서도 동작

### 대안: **E. 클라이언트 전용 HTML**
`.html` 파일 하나를 더블클릭하면 브라우저에서 실행. 설치가 전혀 필요 없는 장점이 있으나, JavaScript(SheetJS)로 Excel 템플릿의 복잡한 양식(상단 메타 행, 병합 셀 등)을 정확히 재현하기 어려움.

## 6.1 자연어 매칭 전략 상세

### 문제 정의

두 가지 매칭이 필요함:

| 단계 | 매칭 대상 A | 매칭 대상 B | 난이도 |
|------|------------|------------|--------|
| ① 사이즈유형 찾기 | 파일 1 상품명의 **기본 부분** | 파일 3 `사이즈유형명` | 중 |
| ② 옵션 슬롯 찾기 | 파일 1 상품명의 **옵션 부분** | 파일 3 `Size01`~`Size30` 값 | 하 |

### 예시로 보는 매칭 흐름

```
입력 상품명: "체이싱래빗 올어바웃 글로우 커버쿠션 # 쿨 라이트"

[Step 0] 전처리 — 정규화
  → "체이싱래빗 올어바웃 글로우 커버쿠션 #쿨 라이트"
     (# 앞뒤 공백 제거, 연속 공백 제거)

[Step 1] 기본 상품명 / 옵션명 분리
  → 기본: "체이싱래빗 올어바웃 글로우 커버쿠션"
  → 옵션: "#쿨 라이트"

[Step 2] 사이즈유형 매칭 (① 기본 상품명 → 사이즈유형명)
  파일 3 사이즈유형명 목록:
    - "체이싱래빗 올어바웃 글로우 커버쿠션 #쿨 라이트" (SD2)
    - ...

  2-a. 정규화 후 포함관계 체크: "체이싱래빗올어바웃글로우커버쿠션" ⊂ "체이싱래빗올어바웃글로우커버쿠션#쿨라이트" → ✅ 매칭
  2-b. 포함관계 실패 시 → RapidFuzz token_sort_ratio 상위 1건 (임계값 80+)
  → 결과: 사이즈유형 = "SD2"

[Step 3] 옵션 슬롯 매칭 (② 옵션명 → Size 값)
  SD2 행의 Size 값: Size01="#쿨 라이트", Size02="#웜 내추럴"

  3-a. 정규화 후 완전 일치: "#쿨라이트" == "#쿨라이트" → ✅ Size01
  3-b. 완전 일치 실패 시 → RapidFuzz ratio 상위 1건 (임계값 75+)
  → 결과: Size01 슬롯
```

### 매칭 알고리즘: 3단계 Fallback

LLM 없이 높은 정확도를 달성하기 위해, **쉬운 방법부터 시도하고 실패 시 점진적으로 고도화**하는 전략을 사용.

```
Level 1 (정규화 후 정확 매칭)
  ↓ 실패 시
Level 2 (정규화 후 포함관계 매칭)
  ↓ 실패 시
Level 3 (RapidFuzz 유사도 매칭)
  ↓ 실패 시
매칭 실패 → 사용자에게 경고 표시
```

#### Level 1: 정규화 후 정확 매칭 (가장 빠르고 확실)
```python
def normalize(text: str) -> str:
    """공백, 특수문자 주변 공백을 제거하여 정규화"""
    text = re.sub(r'\s+', '', text)       # 모든 공백 제거
    text = text.strip().lower()
    return text

# normalize("# 쿨 라이트") == normalize("#쿨 라이트")  → "#쿨라이트" == "#쿨라이트" ✅
```

#### Level 2: 정규화 후 포함관계 매칭
```python
# 기본 상품명이 사이즈유형명에 포함되는지 확인
base_norm = normalize(base_product_name)
for row in option_info:
    if base_norm in normalize(row['사이즈유형명']):
        return row  # 매칭
```

#### Level 3: RapidFuzz 유사도 매칭
```python
from rapidfuzz import fuzz, process

# token_sort_ratio: 토큰 순서를 무시하고 유사도 계산 — 단어 순서가 다를 때 강력
best_match = process.extractOne(
    query=base_product_name,
    choices=[row['사이즈유형명'] for row in option_info],
    scorer=fuzz.token_sort_ratio,
    score_cutoff=80  # 80% 이상만 매칭
)
```

### RapidFuzz를 선택한 이유

| 라이브러리 | 속도 | 한국어 지원 | 매칭 품질 | 비고 |
|-----------|------|:---------:|:--------:|------|
| **RapidFuzz** | ⚡ 매우 빠름 (C++ 기반) | ✅ 유니코드 완전 지원 | 높음 | Levenshtein 기반 + 다양한 scorer |
| difflib | 느림 | ✅ | 보통 | Python 표준 라이브러리 |
| FuzzyWuzzy | 느림 | ✅ | 높음 | RapidFuzz의 구버전, python-Levenshtein 의존 |
| Jamo 분해 | 보통 | ✅ | 특수 케이스에 강함 | 오타 매칭에 유리, 구현 복잡 |

- RapidFuzz는 FuzzyWuzzy의 **10배 이상 빠른** 드롭인 대체제
- `token_sort_ratio`: 토큰 순서 무관 비교 → "쿠션 커버 글로우" vs "글로우 커버 쿠션" 매칭 가능
- `partial_ratio`: 부분 문자열 매칭 → 긴 문자열 속 핵심 부분 추출
- **임계값 미달 시 매칭 실패로 처리** → 잘못된 매칭보다 사용자에게 경고하는 것이 안전

### 정규화 파이프라인 상세

```python
def normalize_for_matching(text: str) -> str:
    """매칭용 텍스트 정규화"""
    text = text.strip()
    text = re.sub(r'\s*#\s*', '#', text)        # "#" 주변 공백 통일: "# 쿨" → "#쿨"
    text = re.sub(r'\s+', ' ', text)             # 연속 공백 → 단일 공백
    text = text.replace('（', '(').replace('）', ')')  # 전각 괄호 → 반각
    return text

def normalize_strict(text: str) -> str:
    """정확 비교용 — 모든 공백/특수문자 차이 무시"""
    text = re.sub(r'[\s\-_·・\(\)（）]', '', text)
    return text.lower()
```

3. `pd.concat`으로 병합
4. (선택) 파일 2가 있으면 88코드 → 1P 상품코드 매칭하여 `상품번호` 채움
5. 출력 컬럼 순서 정리 후 반환

#### 컬럼 매핑 전략
각 파일마다 컬럼명이 다를 수 있으므로, 기존 `_find_col` 정규식 패턴 매칭을 재활용:

| 출력 컬럼 | 입력 파일에서 탐색할 패턴 |
|-----------|--------------------------|
| `지정입고일` | 파일에서 추출 또는 사용자 입력 |
| `브랜드명` | `브랜드명` |
| `88코드` | `88코드`, `바코드` |
| `상품번호` | 파일 2 매칭으로 채움 |
| `상품명` | `상품명` |
| `공급가(VAT 제외)` | `공급가.*VAT.*제외` |
| `정상가(VAT 포함)` | `정상가.*VAT.*포함` |
| `상시행사가(VAT 포함)` | `상시행사가.*VAT.*포함` |
| `발주수량(오프)` | `수량.*오프` |

## 8. 배포 방법

```
배포물: SKU_변환기.exe (단일 파일, ~30-50MB)
```

1. 개발자가 PyInstaller로 `.exe` 빌드
2. 사용자에게 `.exe` 파일 전달 (메일, USB, 사내 공유 등)
3. 사용자는 `.exe` 더블클릭으로 바로 사용 — **설치 과정 없음**
