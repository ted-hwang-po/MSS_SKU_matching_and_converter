## 발견된 문제 (3건)

### 문제 1: `#` 구분자 없는 상품명에서 옵션 분리 실패

**증상**: 옵션 매핑 69건 전부 실패, 사이즈유형/옵션슬롯 모두 None

**원인**: 기존 코드는 상품명에서 `#` 기호를 기준으로 기본 상품명/옵션명을 분리했음.
```
샘플: "체이싱래빗 올어바웃 글로우 커버쿠션 #쿨 라이트" → # 기준 분리 가능
실제: "윙크 파운데이션 팩트 15g 18호" → # 없음, 분리 불가
```

**수정**: 파일 2의 `옵션명` 컬럼을 직접 활용하도록 변경.
- `match_barcode_to_uid()`에서 파일 2의 `옵션명`, `상품명`을 함께 가져옴
- 옵션 매핑 시 파일 1 상품명이 아닌 **파일 2 옵션명**을 사용

### 문제 2: 사이즈유형명 매칭 실패 (브랜드명 접두사 차이)

**증상**: 사이즈유형 매칭 실패 (유사도 부족)

**원인**: 파일 1 상품명과 파일 3 사이즈유형명의 표기 차이.
```
파일 1 상품명:       "윙크 파운데이션 팩트 15g 18호" (브랜드명 없음, 옵션 포함)
파일 3 사이즈유형명: "자빈드서울 윙크 파운데이션 팩트 15g" (브랜드명 포함, 옵션 없음)
```

**수정**: 파일 2의 `상품명`("윙크 파운데이션 팩트 15g", 옵션 없는 기본 상품명)을 사이즈유형 매칭에 우선 사용.
- 파일 2 상품명 → Level 2 포함관계 매칭에서 성공 ("윙크파운데이션팩트15g" ⊂ "자빈드서울윙크파운데이션팩트15g")

### 문제 3: 파일 3에 옵션이 일부만 등록된 경우 슬롯 누락

**증상**: SQ8(아이 쉐이드)는 15개 옵션이지만 파일 3에는 7개만 등록 → 8~15번 옵션 슬롯 매핑 불가

**원인**: 파일 3의 Size01~Size30에 모든 옵션이 등록되지 않음.

**수정**: 파일 2의 UID별 행 순서를 기반으로 슬롯 번호를 자동 배정.
- `_build_uid_option_order()`: 파일 2에서 UID별로 바코드가 등장하는 순서 → Size 슬롯 번호
- 파일 3에 없는 옵션도 순서대로 Size08, Size09, ... 에 자동 배정

## 핵심 아키텍처 변경

**기존 (v1)**: 파일 1 상품명 → `#` 분리 → 옵션 추출 → 파일 3 매칭
**변경 (v2)**: 파일 2 옵션명/상품명 직접 활용 + UID별 행 순서 기반 슬롯 배정

```
[기존] 파일1 상품명 "커버쿠션 #쿨 라이트"
         ↓ # 기준 분리
       기본: "커버쿠션" → 파일3 매칭 → 사이즈유형
       옵션: "#쿨 라이트" → 파일3 Size값 매칭 → 슬롯

[변경] 파일2 상품명 "윙크 파운데이션 팩트 15g" → 파일3 매칭 → 사이즈유형
       파일2 옵션명 "18 커버 라이트" → 파일2 내 순서 → 슬롯 1
       파일2 옵션명 "19 커버 페일"   → 파일2 내 순서 → 슬롯 2
       ...
       (# 분리 방식은 fallback으로 유지)
```
