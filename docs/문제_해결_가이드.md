# SKU 변환기 문제 해결 가이드

---

## 1. 실행 자체가 안 되는 경우

### 증상: 더블클릭해도 아무 반응이 없다
**원인**: Windows Defender 또는 백신이 차단했을 수 있습니다.

**해결 방법**:
1. `SKU_변환기.exe`를 우클릭 → "속성" 클릭
2. 하단에 "이 파일은 다른 컴퓨터에서 왔으며..." 문구가 있으면 → "차단 해제" 체크 → "확인"
3. 다시 더블클릭으로 실행

그래도 안 되면:
- Windows 보안 → 바이러스 및 위협 방지 → 보호 기록에서 차단 여부 확인
- 해당 파일을 "허용된 위협"으로 등록

### 증상: "Windows의 PC 보호" SmartScreen 경고
**해결 방법**:
1. "추가 정보" 클릭
2. "실행" 버튼 클릭

> 이 경고는 Microsoft 인증서가 없는 프로그램에 표시되는 정상적인 알림입니다.

### 증상: "VCRUNTIME140.dll을 찾을 수 없습니다" 오류
**원인**: Visual C++ 런타임이 설치되지 않았습니다.

**해결 방법**:
1. 아래 링크에서 "vc_redist.x64.exe" 다운로드 후 설치
   - https://aka.ms/vs/17/release/vc_redist.x64.exe
2. 설치 후 PC 재시작
3. `SKU_변환기.exe` 다시 실행

## 2. 파일 로딩 오류

### 증상: "파일 1 로드 실패" 또는 "'브랜드명' 컬럼이 없습니다"
**원인**: 파일의 컬럼명이 예상과 다릅니다.

**확인 사항**:
- 파일 1(총 발주 수량)에 `브랜드명` 컬럼이 있는지 확인
- 컬럼명 앞뒤에 불필요한 공백이 있으면 제거
- 엑셀에서 첫 번째 행이 컬럼 헤더인지 확인 (빈 행이 위에 있으면 안 됨)

### 증상: "'바코드' 컬럼을 찾을 수 없습니다" 또는 "'상품코드(UID)' 컬럼을 찾을 수 없습니다"
**원인**: 파일 2(매칭 데이터)의 컬럼명이 예상과 다릅니다.

**확인 사항**:
- 파일 2에 `바코드`, `스타일번호` 컬럼이 있는지 확인
- 상품코드 컬럼은 `상품코드`, `1P 상품코드`, `기존 위탁 상품코드 UID` 중 하나가 있어야 함
- 파일의 첫 번째 행이 완전히 비어있는 경우는 자동 처리되지만, 2행 이상 비어있으면 문제 발생 가능

### 증상: 파일을 선택했는데 브랜드 드롭다운이 비어있다
**원인**: 파일 1의 `브랜드명` 컬럼에 값이 없거나, 잘못된 파일을 선택했을 수 있습니다.

**해결 방법**:
- 올바른 파일(총 발주 수량 데이터셋)을 선택했는지 확인
- 엑셀에서 `브랜드명` 컬럼에 실제 데이터가 있는지 확인

## 3. 매칭 오류

### 증상: "매칭 실패 바코드: [...]" 경고
**원인**: 파일 1의 특정 88코드가 파일 2에 없습니다.

**확인 사항**:
- 표시된 바코드가 파일 2의 `바코드` 컬럼에 존재하는지 확인
- 바코드 앞뒤 공백이나 작은따옴표(') 등 보이지 않는 문자가 있는지 확인
- 파일 2에 해당 상품이 누락된 것이면, 파일 2에 추가 후 다시 실행

### 증상: "옵션 매핑 경고" — 사이즈유형 매칭 실패
**원인**: 상품명으로 파일 3의 사이즈유형명을 찾지 못했습니다.

**확인 사항**:
- 파일 1의 상품명과 파일 3의 사이즈유형명이 유사한지 확인
- 예: 상품명 "글로우 커버쿠션 #쿨 라이트" → 사이즈유형명에 "글로우 커버쿠션"이 포함되어야 함
- 사이즈유형명이 너무 다르면 파일 3을 수정하여 상품명과 유사하게 맞춰주세요

### 증상: "옵션 슬롯 매칭 실패"
**원인**: 상품명의 옵션 부분(# 뒤)이 파일 3의 Size01~Size30 값과 매칭되지 않습니다.

**확인 사항**:
- 상품명: "커버쿠션 **#쿨 라이트**" → 파일 3의 Size01에 "**#쿨 라이트**"가 있어야 함
- 공백 차이는 자동 처리되지만, 완전히 다른 표기(예: "쿨라이트" vs "Cool Light")는 매칭 불가
- 파일 3의 Size 값을 상품명의 옵션 표기와 일치시켜 주세요

## 4. 파일 생성 오류

### 증상: "Permission denied" 또는 파일 저장 실패
**원인**: 출력 파일이 다른 프로그램(엑셀 등)에서 열려 있습니다.

**해결 방법**:
1. 이전에 생성된 출력 파일을 엑셀에서 닫기
2. 다시 "변환 실행" 클릭

### 증상: 출력 파일이 생성되었지만 내용이 비어있다
**원인**: 선택한 브랜드에 해당하는 상품이 없습니다.

**확인 사항**:
- 브랜드 드롭다운에서 올바른 브랜드를 선택했는지 확인
- 파일 1에서 해당 브랜드의 데이터가 존재하는지 엑셀로 직접 확인

---

## 5. 결과 데이터 이상

### 증상: UID 또는 상품번호가 예상과 다르다
**확인 사항**:
- 출력 파일의 UID는 파일 2의 `1P 상품코드` 값을 사용합니다.
- 파일 1의 `상품번호`(위탁 코드)와는 다른 값입니다. 이것은 정상입니다.

### 증상: 가격이 원본과 약간 다르다
**원인**: 소수점 가격은 정수로 반올림 처리됩니다.
- 예: 공급가 6954.545... → 6955
- 이것은 시스템 양식에 맞추기 위한 정상 처리입니다.

### 증상: 옵션 상품의 수량 배분이 잘못되었다
**확인 사항**:
- 파일 3의 Size01, Size02 순서가 올바른지 확인
- 상품명의 옵션 표기가 파일 3의 Size 값과 매칭되는지 확인
- 로그에 옵션 매핑 경고가 있었는지 확인

---

## 6. 프로그램이 멈추거나 응답 없음

### 증상: "변환 실행" 후 프로그램이 멈춘 것 같다
**해결 방법**:
- 데이터가 많은 경우 처리에 시간이 걸릴 수 있습니다. 최대 1분 정도 기다려주세요.
- 로그 영역에 진행 상황이 표시되고 있으면 정상 동작 중입니다.
- 1분 이상 응답이 없으면 프로그램을 닫고 다시 실행해주세요.

## 7. 기타

### 프로그램 업데이트
- 새 버전의 `SKU_변환기.exe` 파일을 받으면 기존 파일을 교체하기만 하면 됩니다.
- 별도의 업데이트나 설치 과정은 없습니다.

### 데이터 보안
- 이 프로그램은 **인터넷에 연결하지 않습니다**.
- 모든 데이터는 사용자의 PC 안에서만 처리됩니다.
- 입력 파일은 읽기만 하며, 원본 파일을 수정하지 않습니다.
