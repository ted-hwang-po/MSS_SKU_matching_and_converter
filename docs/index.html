SKU 변환기

MSS SKU 매칭 및 발주 파일 변환기

시작하기

실행 방법: SKU_변환기.exe 파일을 더블클릭합니다.

SmartScreen 경고가 뜨는 경우
"추가 정보" → "실행" 을 클릭하세요.
Microsoft 인증서가 없는 새 프로그램에 표시되는 정상 알림이며, 보안 문제가 아닙니다.
첫 실행 시 약 5~10초 정도 로딩 시간이 걸릴 수 있습니다.

사용 순서

1

입력 파일 선택

3개의 파일을 각각 "파일 선택" 버튼을 눌러 지정합니다.

순서파일설명
파일 1총 발주 수량 데이터셋브랜드별 전체 발주 정보
파일 2상품코드-바코드 매칭 데이터셋바코드 ↔ 상품코드(UID) 매핑
파일 3상품별 옵션 정보사이즈유형 / 옵션 목록

지원 형식: .xlsx, .csv
파일 1을 선택하면 브랜드 목록이 자동으로 채워집니다.

2

브랜드 선택

두 가지 모드 중 하나를 선택합니다:

모드설명
모든 브랜드파일 1에 있는 모든 브랜드에 대해 일괄 변환합니다.
브랜드 선택목록에서 원하는 브랜드를 선택합니다. Ctrl+클릭으로 여러 개를 동시에 선택할 수 있습니다.
"모든 브랜드"를 선택하면 브랜드별로 각각 2개씩 파일이 생성됩니다.
예: 브랜드 5개 → 총 10개 파일 생성
3

저장 위치 선택

기본값은 바탕화면입니다. 다른 위치를 원하면 "폴더 선택" 버튼을 누르세요.

4

변환 실행

"▶ 변환 실행" 버튼을 클릭합니다. 하단 로그에 진행 과정이 표시되며, 완료되면 "✅ 변환 완료!" 메시지가 나타납니다.

5

결과 확인

"출력 폴더 열기" 버튼으로 저장 폴더를 엽니다.

파일용도
{브랜드명}_시스템업로드_최종파일.xlsx무신사 시스템 업로드용
{브랜드명}_발주리스트_최종파일.xlsx브랜드 송부용 발주 리스트

입력 파일 필수 컬럼

아래 필수 컬럼이 입력 파일에 반드시 있어야 합니다. 컬럼명이 다르거나 누락되면 오류가 발생합니다.
그 외 컬럼은 있어도 무시되므로 삭제하지 않아도 됩니다.

컬럼명설명예시
브랜드명브랜드 필터링 기준자빈드서울

파일 3 — 상품별 옵션 정보

Size 컬럼에 모든 옵션이 등록되지 않아도 파일 2의 순서를 기반으로 자동 배정됩니다.
옵션이 없는 상품(단일 SKU)은 이 파일에 등록하지 않아도 됩니다 (자동으로 MF3 / FREE 처리).

출력 파일 상세

시스템 업로드용 (2개 시트)

시트 1 — 매입상품 구매오더 업로드 양식

  • UID, 스타일번호, 오더 총 수량, 계획 정상가/원가, 사이즈유형이 자동 입력
  • 옵션 상품은 Size01, Size02 등에 옵션별 수량이 배분
  • 납품일, 전자결재 등 나머지 컬럼은 직접 입력 필요

시트 2 — Barcode 업로드(옵션)

  • 스타일번호, UID, 사이즈(옵션명), 바코드가 자동 입력
  • 옵션 없는 상품의 사이즈는 "FREE"

브랜드 송부용 (1개 시트)

  • 브랜드명, 88코드, 상품번호, 상품명, 가격, 총 발주금액, 발주수량 자동 입력
  • 상단에 TOTAL 합계 행 포함
  • 지정입고일은 직접 입력 필요

주의사항

  • 파일 형식 — 반드시 .xlsx 또는 .csv 파일을 사용하세요.
  • 컬럼명 일치 — 입력 파일의 컬럼명이 기존 양식과 동일해야 합니다. 임의 변경 시 오류가 발생할 수 있습니다.
  • 동시 실행 금지 — 변환 중에 "변환 실행" 버튼을 다시 누르지 마세요.
  • 출력 파일 덮어쓰기 — 같은 브랜드로 다시 변환하면 기존 파일이 덮어씌워집니다. 보존이 필요하면 먼저 파일명을 변경하세요.
  • 옵션 매핑 경고 — 로그에 "⚠️ 매핑 경고"가 표시되면 해당 상품의 옵션 정보를 수동으로 확인해야 합니다.

데이터 보안

이 프로그램은 인터넷에 연결하지 않습니다.
모든 데이터는 PC 안에서만 처리되며, 입력 파일은 읽기만 합니다. 원본 파일을 수정하지 않습니다.

기능 B란?

날짜별·브랜드별로 분산된 발주 파일을 하나의 통합 파일로 합칩니다.

통합된 파일은 기능 A(SKU 매칭 변환)의 입력으로 사용하거나, 전체 발주 현황 확인용으로 활용할 수 있습니다.

1. 실행이 안 되는 경우

실행 불가 더블클릭해도 아무 반응이 없다 ▶

원인: Windows Defender 또는 백신이 차단했을 수 있습니다.

해결 방법:

  1. SKU_변환기.exe를 우클릭 → "속성" 클릭
  2. 하단에 "이 파일은 다른 컴퓨터에서 왔으며..." 문구 → "차단 해제" 체크 → "확인"
  3. 다시 더블클릭으로 실행

그래도 안 되면:

  • Windows 보안 → 바이러스 및 위협 방지 → 보호 기록에서 차단 여부 확인
  • 해당 파일을 "허용된 위협"으로 등록
경고 "Windows의 PC 보호" SmartScreen 경고가 뜬다 ▶
  1. "추가 정보" 클릭
  2. "실행" 버튼 클릭
이 경고는 Microsoft 인증서가 없는 프로그램에 표시되는 정상 알림입니다.
실행 불가 "VCRUNTIME140.dll을 찾을 수 없습니다" 오류 ▶

원인: Visual C++ 런타임이 설치되지 않았습니다.

해결 방법:

  1. vc_redist.x64.exe 다운로드 후 설치
  2. PC 재시작
  3. SKU_변환기.exe 다시 실행

2. 파일 로딩 오류

오류 "파일 1 로드 실패" 또는 "'브랜드명' 컬럼이 없습니다" ▶

원인: 파일의 컬럼명이 예상과 다릅니다.

확인 사항:

  • 파일 1(총 발주 수량)에 브랜드명 컬럼이 있는지 확인
  • 컬럼명 앞뒤에 불필요한 공백이 있으면 제거
  • 엑셀에서 첫 번째 행이 컬럼 헤더인지 확인 (빈 행이 위에 있으면 안 됨)
오류 "'바코드' 또는 '상품코드(UID)' 컬럼을 찾을 수 없습니다" ▶

원인: 파일 2(매칭 데이터)의 컬럼명이 예상과 다릅니다.

확인 사항:

  • 파일 2에 바코드, 스타일번호 컬럼이 있는지 확인
  • 상품코드 컬럼은 상품코드, 1P 상품코드, 기존 위탁 상품코드 UID 중 하나가 있어야 함
  • 첫 번째 행이 완전히 비어있는 경우는 자동 처리되지만, 2행 이상 비어있으면 문제 발생 가능
경고 파일을 선택했는데 브랜드 목록이 비어있다 ▶

원인: 파일 1의 브랜드명 컬럼에 값이 없거나, 잘못된 파일을 선택했을 수 있습니다.

  • 올바른 파일(총 발주 수량 데이터셋)을 선택했는지 확인
  • 엑셀에서 브랜드명 컬럼에 실제 데이터가 있는지 확인
참고 "브랜드 선택" 모드에서 여러 개를 선택하는 방법 ▶
  • Ctrl + 클릭: 개별 브랜드를 하나씩 추가/해제
  • Shift + 클릭: 범위 선택 (처음 클릭한 항목부터 마지막 클릭한 항목까지)

3. 매칭 오류

경고 "매칭 실패 바코드" 경고 ▶

원인: 파일 1의 특정 88코드가 파일 2에 없습니다.

  • 표시된 바코드가 파일 2의 바코드 컬럼에 존재하는지 확인
  • 바코드 앞뒤 공백이나 보이지 않는 문자가 있는지 확인
  • 파일 2에 해당 상품이 누락됐다면, 추가 후 다시 실행
경고 "옵션 매핑 경고" — 사이즈유형 매칭 실패 ▶

원인: 상품명으로 파일 3의 사이즈유형명을 찾지 못했습니다.

  • 파일 1의 상품명과 파일 3의 사이즈유형명이 유사한지 확인
  • 예: 상품명 "글로우 커버쿠션 #쿨 라이트" → 사이즈유형명에 "글로우 커버쿠션"이 포함되어야 함
  • 너무 다르면 파일 3을 수정하여 유사하게 맞춰주세요
경고 "옵션 슬롯 매칭 실패" ▶

원인: 상품명의 옵션 부분(# 뒤)이 파일 3의 Size 값과 매칭되지 않습니다.

  • 상품명: "커버쿠션 #쿨 라이트" → 파일 3의 Size01에 "#쿨 라이트"가 있어야 함
  • 공백 차이는 자동 처리되지만, 완전히 다른 표기는 매칭 불가
  • 파일 3의 Size 값을 상품명의 옵션 표기와 맞춰주세요

4. 파일 생성 오류

오류 "Permission denied" 또는 파일 저장 실패 ▶

원인: 출력 파일이 엑셀 등 다른 프로그램에서 열려 있습니다.

  1. 이전에 생성된 출력 파일을 엑셀에서 닫기
  2. 다시 "변환 실행" 클릭
경고 출력 파일이 생성되었지만 내용이 비어있다 ▶

원인: 선택한 브랜드에 해당하는 상품이 없습니다.

  • 브랜드 드롭다운에서 올바른 브랜드를 선택했는지 확인
  • 파일 1에서 해당 브랜드의 데이터가 있는지 엑셀로 확인

5. 결과 데이터 이상

참고 UID 또는 상품번호가 예상과 다르다 ▶
  • 출력 파일의 UID는 파일 2의 1P 상품코드 값을 사용합니다.
  • 파일 1의 상품번호(위탁 코드)와는 다른 값이며, 이것은 정상입니다.
참고 가격이 원본과 약간 다르다 ▶

소수점 가격은 정수로 반올림 처리됩니다.

예: 공급가 6954.545... → 6955

이것은 시스템 양식에 맞추기 위한 정상 처리입니다.

경고 옵션 상품의 수량 배분이 잘못되었다 ▶
  • 파일 3의 Size01, Size02 순서가 올바른지 확인
  • 상품명의 옵션 표기가 파일 3의 Size 값과 매칭되는지 확인
  • 로그에 옵션 매핑 경고가 있었는지 확인

6. 프로그램이 멈추거나 응답 없음

경고 "변환 실행" 후 프로그램이 멈춘 것 같다 ▶
  • 데이터가 많으면 처리에 시간이 걸릴 수 있습니다. 최대 1분 정도 기다려주세요.
  • 로그 영역에 진행 상황이 표시되고 있으면 정상 동작 중입니다.
  • 1분 이상 응답이 없으면 프로그램을 닫고 다시 실행해주세요.

8. 발주 파일 통합 (기능 B) 관련

오류 "헤더를 찾을 수 없습니다" 오류 ▶

원인: 발주 파일에 브랜드명 컬럼이 있는 행을 찾지 못했습니다.

  • 파일의 상단 1~9행 사이에 브랜드명이 포함된 헤더 행이 있어야 합니다
  • 기존 브랜드 송부용 파일 양식과 동일한 구조여야 합니다
  • 헤더 행 위에 빈 행이 많으면(10행 이상) 인식 실패할 수 있습니다
경고 상품번호가 채워지지 않는다 ▶

원인: 매칭 파일이 없거나, 바코드가 매칭되지 않았습니다.

  • 매칭 파일을 선택했는지 확인
  • 매칭 파일에 바코드 컬럼과 신규 상품코드(또는 1P 상품코드, 상품코드) 컬럼이 있는지 확인
  • 발주 파일의 88코드 값이 매칭 파일의 바코드 값과 일치하는지 확인
경고 지정입고일이 비어있다 ▶

원인: 발주 파일 상단에 "지정입고일" 텍스트와 날짜 정보를 찾지 못했습니다.

  • 파일의 1~5행 사이에 "지정입고일"이라는 텍스트가 있고, 바로 옆 셀에 날짜(예: 4/1(수))가 있어야 합니다
  • 없는 경우 통합 후 엑셀에서 직접 입력해주세요
