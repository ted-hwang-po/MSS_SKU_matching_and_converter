# MSS SKU 매칭 및 발주 파일 변환기 — UX 리뷰 보고서

> 작성일: 2026-03-29
> 분석 대상: app.py (tkinter GUI), core/ 모듈 전체

---

## 1. 개요

본 문서는 MSS SKU 매칭 및 발주 파일 변환기의 사용자 경험(UX)을 **PM / Designer / Engineer** 3가지 관점에서 분석하고, 구체적인 개선 방안을 제시한다.

### 1.1 분석 범위
- 전체 GUI 레이아웃 및 인터랙션 (`app.py`)
- 파일 입출력 흐름 (`core/loader.py`, `core/generator.py`, `core/merger.py`)
- 에러 처리 및 사용자 피드백 (`core/matcher.py`)
- 데이터 처리 흐름의 사용자 가시성

### 1.2 대상 사용자
- 비개발자 업무 담당자 (이커머스/리테일 운영팀)
- 매일 반복적으로 발주 데이터를 처리하는 실무자
- Windows PC 사용 (PyInstaller .exe 배포)

## 2. 현재 UX 문제점 분석

### 2.1 심각도 높음 (High)

| # | 이슈 | 위치 | 영향 | 상세 |
|---|------|------|------|------|
| H1 | 프로그레스 바 없음 | `app.py:148-155` | 브랜드 10개 이상 처리 시 "멈춘 건가?" 불안감 | 텍스트 로그만 존재하며, 전체 진행률을 한눈에 파악 불가 |
| H2 | 파일 유효성 검증 없음 | `app.py:357-364` | 잘못된 파일 선택 시 실행 후에야 에러 발견 | 3개 파일 모두 선택 후 "실행" 버튼을 눌러야만 오류 확인 가능 |
| H3 | 브랜드 체크박스 영역 100px 고정 | `app.py:115` | 브랜드 20개 이상 시 스크롤이 과도하게 필요 | `Canvas height=100`으로 고정, 한 번에 4-5개만 표시 |
| H4 | Python traceback 에러 노출 | `app.py:404,410` | 비개발자에게 공포감, 무엇이 잘못인지 파악 불가 | `traceback.format_exc()` 그대로 로그에 출력 |
| H5 | 출력 파일 덮어쓰기 확인 없음 | `app.py:396-399` | 이전 작업 결과를 실수로 덮어쓸 위험 | `Path.exists()` 체크 없이 바로 저장 |

| # | 이슈 | 위치 | 영향 | 상세 |
|---|------|------|------|------|

| M3 | 실행 취소/리셋 없음 | 전반 | 파일을 잘못 선택해도 개별 초기화 불가 | 앱을 재시작해야 초기 상태로 복귀 |
| M4 | 세션 간 파일 경로 미기억 | 전반 | 앱 재시작 시 매번 3개 파일 재선택 필요 | 매일 같은 파일 조합을 사용하는 경우 큰 마찰 |
| M5 | 툴팁/도움말 없음 | 전반 | "파일 1이 뭐지?", "사이즈유형이 뭐지?" 등 혼란 | 각 입력 필드에 대한 설명이 없음 |

## 3. 개선 아이디어 — PM 관점 (비즈니스 가치 & 고객 임팩트)

### P1. 작업 이력 대시보드
- **설명**: 이전 변환 기록(날짜, 브랜드, 성공/실패, 파일 수)을 저장하고 조회
- **근거**: 반복 작업에서 "지난번에 어떤 브랜드를 처리했는지" 확인 필요
- **구현**: JSON 파일에 이력 저장, UI에 이력 보기 버튼 추가
- **검증 가정**: 사용자가 실제로 이전 작업을 참조하는 빈도

### P2. 출력 미리보기
- **설명**: 변환 실행 전 "이런 파일이 생성됩니다" 요약을 다이얼로그로 표시
- **근거**: 실행 전 확인 단계를 통해 실수 방지
- **구현**: 브랜드별 예상 행 수, 출력 파일명, 예상 파일 수 표시
- **검증 가정**: 잘못된 브랜드 선택으로 인한 재작업 빈도

### P3. 일괄 재실행 프리셋
- **설명**: 자주 사용하는 파일 조합 + 브랜드 선택을 "프리셋"으로 저장/로드
- **근거**: 매일 같은 3개 파일 + 같은 브랜드 조합 → 매번 재설정은 큰 마찰
- **구현**: 프리셋 이름 지정, 저장/로드/삭제 UI
- **검증 가정**: 동일 파일 조합 반복 사용 빈도

### P4. 매칭 실패 리포트 내보내기
- **설명**: 옵션 매칭 경고, 미매칭 바코드를 별도 Excel 파일로 내보내기
- **근거**: 후속 수동 처리에 필요한 정보를 구조화하여 제공
- **구현**: 경고 데이터를 DataFrame으로 구성, Excel 저장
- **검증 가정**: 매칭 실패 건의 후속 처리 프로세스 존재 여부

### P5. 파일 덮어쓰기 보호
- **설명**: 이미 존재하는 출력 파일이 있으면 확인 다이얼로그 표시
- **근거**: 이전 작업 결과를 실수로 날리는 사고 방지
- **구현**: `Path.exists()` 체크 + messagebox.askyesno
- **검증 가정**: 같은 브랜드를 여러 번 재실행하는 빈도

## 4. 개선 아이디어 — Designer 관점 (사용성 & UX)

### D1. 시각적 진행 상태 표시
- **설명**: ttk.Progressbar + "3/10 브랜드 처리 중..." 상태 텍스트
- **근거**: 텍스트 로그만으로는 전체 진행률을 직관적으로 파악 불가
- **구현**: determinate 모드 프로그레스 바, 현재 처리 중인 브랜드명 표시
- **영향**: 처리 중 불안감 해소, 예상 완료 시점 파악 가능

### D2. 파일 상태 인디케이터
- **설명**: 파일 선택 직후 유효성 검증 → 초록/빨강 상태 아이콘 표시
- **근거**: "실행" 버튼 누르기 전에 파일이 올바른지 확인 가능
- **구현**: 파일 선택 시 비동기 검증 (필수 컬럼 존재 여부), 상태 라벨 표시
- **영향**: 잘못된 파일 선택으로 인한 시간 낭비 방지

### D3. 브랜드 선택 영역 확대 + 칩 UI
- **설명**: 체크박스 영역 100px → 180px, 선택된 브랜드를 상단에 태그 형태로 표시
- **근거**: 현재 4-5개만 표시되어 스크롤이 과도하게 필요
- **구현**: Canvas height 증가, 선택된 브랜드 요약 영역 개선
- **영향**: 브랜드 선택 시 가시성 및 조작성 대폭 개선

### D4. 에러 메시지 사용자 친화적 변환
- **설명**: Python traceback 대신 "파일 2에서 '바코드' 컬럼을 찾을 수 없습니다" 같은 안내
- **근거**: 비개발자에게 traceback은 의미 없는 공포감만 유발
- **구현**: 예외 유형별 한국어 메시지 매핑, traceback은 디버그 모드에서만 표시
- **영향**: 사용자가 스스로 문제를 해결할 수 있는 가능성 증가

### D5. 드래그앤드롭 파일 입력
- **설명**: 파일 선택 영역에 드롭존 표시, 파일 탐색기에서 끌어다 놓기 지원
- **근거**: 반복 작업 시 클릭 횟수 감소, 직관적인 인터랙션
- **구현**: tkinterdnd2 활용 (선택적 의존성, 미설치 시 버튼만 표시)
- **영향**: 파일 입력 속도 향상, 현대적 UX 제공

## 5. 개선 아이디어 — Engineer 관점 (기술적 가능성 & 효율)

### E1. 세션 상태 자동 저장
- **설명**: 마지막 사용한 파일 경로, 저장 위치, 브랜드 선택을 JSON에 자동 저장
- **근거**: 매일 같은 파일을 쓰는 사용자에게 매번 3개 파일 재선택은 큰 마찰
- **구현**: `~/.mss_converter/session.json`에 저장, 앱 시작 시 자동 복원
- **영향**: 앱 재시작 후 즉시 작업 재개 가능

### E2. 파일 로드 시 스키마 검증
- **설명**: 파일 선택 직후 필수 컬럼 존재 여부를 검증하고 결과 표시
- **근거**: 현재는 "실행" 후에야 컬럼 부재 에러 발생
- **구현**: 파일 로드 함수 호출 후 컬럼 체크, 상태 라벨에 결과 표시
- **영향**: 잘못된 파일 조기 발견으로 시간 절약

### E3. MouseWheel 바인딩 스코프 수정
- **설명**: `bind_all` → 체크박스 캔버스 위에서만 동작하도록 변경
- **근거**: 현재 로그 영역 위에서 마우스 휠을 굴려도 체크박스가 스크롤됨
- **구현**: `bind_all` 제거, Canvas와 내부 프레임에만 바인딩
- **영향**: 스크롤 동작의 예측 가능성 향상

### E4. 창 크기 반응형 전환
- **설명**: `resizable(True, True)` + 최소 크기 설정
- **근거**: 브랜드가 많거나 로그가 길 때 창을 키워서 한눈에 볼 수 있음
- **구현**: `minsize(720, 800)`, weight 설정으로 확장 시 로그 영역 자동 확장
- **영향**: 다양한 모니터 해상도에 대한 적응성 향상

### E5. 비동기 파일 로딩 + 취소 기능
- **설명**: 처리 중 "취소" 버튼으로 긴 작업 중단 가능
- **근거**: 잘못된 설정으로 실행 시 완료까지 기다려야 함
- **구현**: threading.Event 기반 취소 플래그, 브랜드 루프에서 체크
- **영향**: 실수 시 빠른 복구, 사용자 제어감 향상

## 8. 사용성 인터뷰 스크립트

위 가정 검증을 위한 인터뷰 가이드 (The Mom Test 원칙 준수).

### Opening (2-3분)
> "안녕하세요, 저희 SKU 매칭 도구를 더 편하게 만들고 싶어서 이야기를 듣고 있습니다. 정답이 없으니 편하게 말씀해주세요."

### Warm-Up (5분)
- "업무에서 어떤 역할을 맡고 계세요?"
- "발주 데이터를 다루신 지 얼마나 되셨나요?"
- "하루에 이 작업을 몇 번 정도 하시나요?"

### Core: 구체적 경험 탐색 (15-20분)

**최근 작업 흐름:**
- "가장 최근에 이 도구를 사용했던 때를 떠올려보세요. 처음부터 끝까지 어떻게 진행하셨나요?"
- "파일 3개를 준비하는 과정은 어떤가요? 어디서 받아오시나요?"
- "브랜드를 선택할 때 보통 몇 개를 고르시나요?"

**어려웠던 점:**
- "이 도구를 쓰면서 가장 답답했던 순간이 있었나요? 어떤 상황이었나요?"
- "에러가 나면 어떻게 하셨나요? 누구한테 물어보셨나요?"
- "결과 파일이 잘못 나온 적이 있나요? 어떻게 알아차리셨나요?"

**해결 시도:**
- "이 도구 없이 작업하던 때는 어떻게 하셨나요?"
- "지금 이 작업에서 가장 시간이 많이 걸리는 부분은 뭔가요?"
